# Goal

在 REQ-032 Envelope 基础上，落实 Thread / Correlation 的创建与传播规则，
使一个 REQ 的完整协作链路（Pandas↔Menglan↔Huahua 多轮往返）可以被追踪和重建。

# In Scope

- `scripts/pandas-heartbeat.sh`：`auto_claim()` 在路由第一个 request 时创建 `thread_id`
  - 格式：`thread_{req_id}_{epoch}`
  - 持久化：将 `thread_id` 写入 REQ 文件的 `Agent Notes`（供后续心跳读取复用）
- `scripts/pandas-heartbeat.sh`：每次发出新 request（同一任务）复用已有 `thread_id`，创建新 `correlation_id`
  - correlation_id 格式：`corr_{req_id}_{epoch}`，每次新 request 刷新 epoch
- `scripts/pandas-heartbeat.sh`：处理 response 时，提取 `correlation_id`，确认与发出的 request 配对
  - 配对失败（correlation_id 不匹配）→ warn + 写入 `failed/`
- `_handle_dev_complete()` / `_handle_tc_complete()` 等 handler：从 response 消息读取 `thread_id` / `correlation_id` 并写入下一个 request（链路续接）
- 新增辅助函数 `thread_get_or_create <req_id>`：读取 REQ 文件 Agent Notes 中的 thread_id，不存在则创建并写入
- 新增辅助函数 `correlation_new <req_id>`：生成新 correlation_id

# Out of Scope

- UI 层的 thread 链路可视化
- 跨 Team 的 thread（本 REQ 只覆盖 Pandas Team 内部）
- done/ 目录的 thread_id 索引文件（后续可单独加）

# Acceptance Criteria

- `auto_claim()` 路由第一个 request 时，REQ 文件 Agent Notes 中出现 `thread_id: thread_{req_id}_{epoch}`
- 后续同一 REQ 的所有 request 携带相同 `thread_id`，不同 `correlation_id`
- response 中携带 `correlation_id`，handler 提取后验证与发出的 request correlation_id 匹配
- correlation_id 不匹配时 → warn 日志 + 消息移到 failed/
- `thread_get_or_create` 函数幂等：对同一 req_id 多次调用返回相同 thread_id
- `npm run build` 通过
- `npm test` 通过，覆盖 thread 创建、复用、correlation 校验场景

# Test Case Design Notes

需覆盖以下场景：

- TC-034-01：首次 request — thread_id 创建，写入 REQ Agent Notes
- TC-034-02：第二次 request（同一 REQ）— 复用已有 thread_id，新 correlation_id
- TC-034-03：response 配对 — correlation_id 匹配 → handler 正常执行
- TC-034-04：response correlation_id 不匹配 → warn + 消息移到 failed/
- TC-034-05：`thread_get_or_create` 幂等性 — 两次调用返回相同值
- TC-034-06：完整链路重建 — 给定 thread_id，在 done/ 中 grep 可找到所有相关消息

## thread_id 持久化位置

thread_id 写入 REQ 文件的 Agent Notes 区域，格式：

```markdown
# Agent Notes

## ATM Threading
thread_id: thread_REQ-030_1710867000
last_correlation_id: corr_REQ-030_1710867120
```

使用 `awk` / `sed` 读写，避免破坏其他 Agent Notes 内容。

## 链路重建示例

```bash
# 给定 thread_id，从 done/ 重建完整链路
thread_id="thread_REQ-030_1710867000"
grep -rl "thread_id: ${thread_id}" ~/shared-resources/inbox/for-*/done/ | sort
```
